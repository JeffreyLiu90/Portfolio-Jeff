Ramen Empire

Order delicious Ramen from the comfort of wherever you are!

.

Tech

HTML / CSS / JS / Node.js / jQuery / Ajax / Express / Twilio API / Google Map API

Project Summary

Ramen Empire is a full-stack food ordering web application, where the user can order food items from the comfort of where ever they are located. The user is able to check out once they ordered their items, and receive a text that tells them how long it will take for the food to be prepared. On the last page, a confirmation number will be provided, and a Google Map indicating the distance between the user and the restaurant, and the duration it'd take for pick up based on different transportation methods, such as walking, biking or driving.

The purpose of this project was to create a full-stack clone of the famous food ordering app called Ritual. As the concept of ordering food online has become popular recently, my team decided that learning how to build this type of application would be useful and beneficial for our own learning as well.


Features

  • Order food item from the menu list and its quantity
  • Insert personal and bank information on the payment page for the order
  • Receive a text indicating the food preparation time
  • View Google map indicating the distance from the user's location to the restaurant
  • View the duration and direction to the restaurant via walking, biking or driving

Challenges

As a learning developer with 3 weeks of programming experience, this was the first Full-Stack web app that I built alongside with 3 other team members. It was all of our first times working in a group setting in this field, so navigating through the planning process was new to us. We had to complete this project in a 1 week time, so there was added pressure time crunch pressure. We faced many obstacles along the way, such as setbacks from learning new technologies such as Twilio, communicating with team members and Git merge conflicts.

Although the first leg of the project was challenging due to our inexperience, we quickly adapted, learned from our errors and successfully completed the project on time. Communication improved and help was provided to each other whenever needed.
