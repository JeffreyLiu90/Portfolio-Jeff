Ramen Empire

Order delicious Ramen from the comfort of wherever you are!

.

Tech

HTML / CSS / JS / Node.js / jQuery / Ajax / Express / Twilio API / Google Map API

Project Summary

Ramen Empire is a full-stack food ordering web application, where the user can order food items from the comfort of where ever they are located. The user is able to check out once they ordered their items, and receive a text that tells them how long it will take for the food to be prepared. On the last page, a confirmation number will be provided, and a Google Map indicating the distance between the user and the restaurant, and the duration it'd take for pick up based on different transportation methods, such as walking, biking or driving.

The purpose of this project was to create a full-stack clone of the famous food ordering app called Ritual. As the concept of ordering food online has become popular recently, my team decided that learning how to build this type of application would be useful and beneficial for our own learning as well.


Features

  • Order food item from the menu list and its quantity
  • Insert personal and bank information on the payment page for the order
  • Receive a text indicating the food preparation time
  • View Google map indicating the distance from the user's location to the restaurant
  • View the duration and direction to the restaurant via walking, biking or driving

Challenges

As a learning developer with 3 weeks of programming experience, this was the first Full-Stack web app that I built alongside with 3 other team members. It was all of our first times working in a group setting in this field, so navigating through the planning process was new to us. We had to complete this project in a 1 week time, so there was added pressure time crunch pressure. We faced many obstacles along the way, such as setbacks from learning new technologies such as Twilio, communicating with team members and Git merge conflicts.

Although the first leg of the project was challenging due to our inexperience, we quickly adapted, learned from our errors and successfully completed the project on time. Communication improved and help was provided to each other whenever needed.

Tweeter

A Twitter clone with lots of GIFS

.

Tech

HTML / CSS / JS / Node.js / jQuery / Ajax / Express

Project Summary

Tweeter is a clone of the popular social media app Twitter. It is Ajax based, and uses jQuery, HTML5, CSS 3. The purpose of this project was to create a single page that loads and runs data asynchronously, meaning it will not have to reload once new data/tweet is inputted.

Features

  • User can write a tweet or message to express what is on their mind
  • Once a tweet or message is written, it shows the user's avatar, and time of post by days
  • User cannot write a message that exceeds 140 characters, which is the maximum length
  • User can scroll back to the top with an up arrow button that appears once the user scrolls down the page to a certain point

Challenges

First introduction to Ajax and jQuery was through building this Twitter clone. Initially, it was a bit difficult to implement as my vanilla JavaScript skills were not as strong. HTML and CSS was also not as strong, so with 4 days to build this project, it was quite challenging. However, by the end of this project, I became much more comfortable with each concept, and jQuery became much more straightforward.

Kitchen Hero

The Recipe app that can help transform you into the ultimate home chef

.

Tech

PostgreSQL / Express / Node.js / ReactJS/ Material-UI / Spoonacular API / Clarifai API / Auth0 API / Google Translate API

Project Summary

KitchenHero is a Full-stack web application that allows users to search for both food and cocktail recipes based on the ingredients that they have by either typing them in the search bar, or by uploading a picture of their food product.

This app was created during the beginning of the Covid pandemic, when the lock-down and social-distancing rules were strict, resulting in most people having to stay at home. This also resulted in more people cooking at home, instead of eating out.

This app was designed for not only people that have experience cooking, but also individuals who are beginners. Instead of scavaging through countless websites for different recipes, this app solves this problem, and makes best use of any left over ingredients in the pantry.

Features

  • Search recipe based on the ingredient or image upload of the food item
  • View recipe information in modal, such as instructions, ingredients, etc.
  • Share recipe on Facebook or translate to another language
  • Save the recipe to user's Saved Recipes
  • Create a custom recipe and save it to User's Recipe's
  • Keep track of inventory and their expiry dates

Challenges

As a learning developer, this was the first project that was done completely remote with two other team mates due to Covid, which was quite challenging. We had 1.5 weeks to complete this project and at the beginning of the project, most of us were very new and inexperienced with React, let alone building a full-stack application with new technologies. However, we were able to overcome the challenges by finishing the project on time, and through consistent communication via Zoom and Slack. We planned extensively, and had great team chemistry.
